# VeryGood Redirect Policy Draft

목적

현재 허브 구조를 유지한 채, 기존 경로를 언제까지 보존하고 무엇을 장기적으로 흡수하거나 리다이렉트할지 정리한다.
이번 문서는 정책 초안이며, 실제 리다이렉트는 아직 적용하지 않는다.

기본 원칙

- 기존 주문, 결제, 법률, 관리자 기능은 즉시 건드리지 않는다.
- 허브에서 충분한 대체 콘텐츠가 준비된 뒤에만 흡수 또는 리다이렉트를 검토한다.
- SEO 영향이 있는 경로는 `noindex`, canonical, 내부 링크 정리까지 함께 판단한다.
- `/en/...` 경로는 아래 정책을 동일하게 미러 적용한다.

## Absorption Priority

1. `/home-legacy`
   새 허브가 충분히 안정화되면 가장 먼저 리다이렉트 후보가 될 수 있다.

2. `/about`, `/business-story`
   `/brand`의 스토리 구조가 더 풍부해진 뒤 흡수 검토가 가능하다.

3. 외부 한국어 매장 자산
   `vcc체험단페이지` 기반의 매장 정보, 지도, 이미지 자산은 `/store`로 단계적으로 흡수한다.

## Redirect Timing Guide

- 1단계
  `/home-legacy` 유지 + `noindex` 지속
- 2단계
  `/brand`에 기존 소개/스토리 내용이 충분히 이관되면 `/about`, `/business-story` 리다이렉트 검토
- 3단계
  매장 자산 흡수가 끝나면 외부 한국어 매장 진입 경로를 `/store` 중심으로 재정리

## Before Any Redirect

- 대응 페이지의 정보량이 기존 페이지와 비교해 부족하지 않은가
- 내부 링크와 메뉴가 새 경로를 기준으로 정리되어 있는가
- canonical, sitemap, `noindex` 정책을 함께 손볼 준비가 되었는가
- 외부 유입이 많은 경로인지 먼저 확인했는가
- 주문, 결제, 관리자, 법률 페이지처럼 기능성 경로를 잘못 묶지 않았는가

## Current Recommendation

- 지금 즉시 리다이렉트할 경로는 없다.
- 가장 먼저 검토할 후보는 `/home-legacy -> /`.
- `/about`, `/business-story`는 지금 당장 옮기지 말고 `/brand` 카피와 콘텐츠가 더 쌓인 뒤 다시 판단한다.
